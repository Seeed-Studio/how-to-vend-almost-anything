# Layer 1 — The Idea System

*Fabrication as local infrastructure.*

Fab Labs began with a single principle: access to tools is access to production. When that access is distributed rather than centralized, capability follows. A community no longer has to depend on distant factories to make what it needs; it can make locally, at the scale and pace the situation demands.

Seen this way, a Fab Lab is not merely a workshop. It is a **micro-infrastructure node** — a point in a network where raw capability is turned into useful outcomes for the people nearby.

This project starts from that premise and asks a deliberately broad question:

> How can fabrication practices be used to design systems that "vend almost anything" at a local scale?

The word *vending* is chosen on purpose. It is not about beverage machines. It is a compact model for a much larger pattern — **authenticate, order, control, dispense** — that recurs across almost every act of local production and exchange.

## Three directions

The idea layer explores that question along three lines.

- **Skills that serve local needs.** A lab is most valuable when it is coupled to its surroundings — responding to real constraints and real people rather than to abstract production targets.
- **Complexity broken into modules.** Large systems become approachable when decomposed into parts that can be understood and replaced independently. Vending is an ideal teaching case because it decomposes so cleanly: identity, ordering, control, and dispensing.
- **Value created locally.** Instead of optimizing for scale, the emphasis shifts to accessibility, adaptability, and immediate usefulness — value measured by proximity to need. And when a good local design is kept open and shareable, it becomes everyone's starting point rather than a one-off.

## What this layer is (and is not)

At this stage the vending machine is not yet hardware. It is a **way of thinking** about how local production systems can be structured. The physical machine that follows is an instrument for testing this idea — not the idea itself.

The remaining layers make the idea concrete: a working reference system (Layer 2), a generalized architecture (Layer 3), a framework for replication (Layer 4), and the economic value it unlocks (Layer 5).
