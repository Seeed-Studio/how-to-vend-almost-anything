# Layer 5 — Real World Value

*From machines to local economies.*

At scale, the question is no longer *how the machine works*, but *why it matters*. A vending system, in this context, is not a device. It is a **micro-infrastructure node within a local economy** — a small, autonomous point where supply meets demand with very little friction.

## Four perspectives

- **The convenience-store model, distributed.** A vending system can be read as a distributed retail node. It shortens the distance between what a community has and what it needs, and it does so without a storefront, staff, or fixed hours.
- **Efficiency measured as accessibility.** Efficiency here is not only cost or speed. The primary metric is how close a service can be brought to the point of need. A machine that is *present* often beats a warehouse that is *cheaper*.
- **Scale without centralization.** Growth does not require a bigger central facility. It requires more nodes. Many small, independent machines can compose into a distributed network of fabrication and distribution.
- **Fab Labs as network components.** The broadest implication: Fab Labs are not isolated educational spaces. Each one can be a live component of a distributed production-and-consumption network.

## The larger vision

This project begins with a vending machine, but it is not limited to one. The machine is a **structural example** used to explore a larger question:

> How can fabrication systems be designed to operate as local infrastructure?

The answer is not a single device but a layered system — from ideas, to a reference implementation, to a generalized design, to a framework for replication, and finally to real-world economic value.

In that structure, "how to vend almost anything" is less a slogan than a **framework for thinking about distributed manufacturing systems**.

The vending machine is only the starting point.
